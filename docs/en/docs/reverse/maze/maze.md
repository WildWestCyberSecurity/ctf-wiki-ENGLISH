# Maze Problems

Maze problems have the following characteristics:

* A "map" is laid out in memory
* User input is limited to a few characters.
* Generally there is only one maze entrance and one maze exit

The map can be composed of printable characters (such as `#` and `*`) (this is very obvious — checking strings basically tells you it's a maze problem), or it can be represented purely with non-printable hexadecimal values. The map can be arranged as one very long string, or laid out line by line. If laid out line by line, since mazes are generally quite large, the functions used to arrange the maze by row (note that the arrangement is not in sequential order — each row corresponds to a specific row number, and you need to determine the row numbers to restore the maze map) will obviously repeat many times.

The restricted characters are usually easy-to-remember combinations (if not, there's nothing you can do about it), such as `w/s/a/d`, `h/j/k/l`, `l/r/u/d`, and similar combinations. Of course, the specific operation of each key needs to be determined through analysis (for those using a single string to represent the maze, you could use the `t` key to mean moving right by `12` characters, for example). For two-dimensional maps, the author generally sets an `X coordinate` and a `Y coordinate` to save the current position. We can also use this characteristic as a starting point for analysis.

In general, a maze has only 1 entrance and 1 exit, such as the entrance at the top-left corner `(0, 0)` and the exit at the bottom-right corner `(max_X, max_Y)`. But it's also possible that the exit is at the center of the maze, represented by a `Y` character, etc. The conditions for solving a maze problem also need to be determined based on the specific situation.

Of course, there may be more than one path through the maze. There could be multiple paths, but a specific one is required, for example the one with the minimum cost. This can then be transformed into an algorithm problem.

## Volga Quals CTF 2014: Reverse 100

Next, let's use this challenge as an example. This is a simple maze problem. The map is laid out by row in shuffled order, using the characters `#` and `*`.

The corresponding `crackme` can be downloaded here: [rev100](https://github.com/ctf-wiki/ctf-challenges/blob/master/reverse/maze/2014_volga_quals/rev100)

Through debugging, we can determine that after each user input of `L/R/U/D`, the program first prints your input, then prints the corresponding `X/Y coordinates`. The final success condition is when `pos_x == 89 && pos_y == 28`. Based on the above information, we can obtain the path to exit the maze.
